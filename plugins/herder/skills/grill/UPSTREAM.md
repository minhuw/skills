# Upstream provenance

This skill adapts the interview and domain-modeling workflow from [mattpocock/skills](https://github.com/mattpocock/skills), especially `skills/productivity/grilling/SKILL.md`, `skills/engineering/grill-with-docs/SKILL.md`, and `skills/engineering/domain-modeling/SKILL.md`, at commit `e9fcdf95b402d360f90f1db8d776d5dd450f9234`, imported on 2026-07-15.

Herder keeps the one-question interview, factual investigation, glossary discipline, terminology challenges, edge-case scenarios, and selective ADR criteria. It replaces upstream issue-tracker setup, labels, and publishing with creation of one canonical Herder plan. It also records accepted `CONTEXT.md` and ADR work in the plan instead of mutating uncommitted project documentation during the interview, so Fire receives the decisions and commits documentation with implementation. Existing-plan refinement remains available through `--plan`.
